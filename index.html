    SMART BTQ

    Sistem Monitoring Baca Tulis Al-Qur’an Berbasis Data

    Tentang Aplikasi

    SMART BTQ adalah platform monitoring Baca Tulis Al-Qur’an berbasis data yang digunakan untuk analisis perkembangan siswa secara real-time, terstruktur, dan terintegrasi antar sekolah.

    Lingkungan Sistem Produksi
    Status Sistem Aktif
    Sumber Data Database Terintegrasi

    Petunjuk Penggunaan

    Struktur Menu

    Menu pada sistem SMART BTQ terdiri dari:

    • Ringkasan Eksekutif
      Menampilkan gambaran umum performa BTQ pada periode tertentu.
    • Data Sekolah
      Menyajikan data detail setiap sekolah secara lengkap.
    • Analisis Grafik
      Menampilkan visualisasi data dalam bentuk grafik untuk memudahkan pemahaman.
    • Analisis Tren
      Menyajikan analisis perkembangan data dalam periode lebih dari satu bulan.
    • Kesimpulan
      Menyediakan kesimpulan dan saran berdasarkan hasil analisis data.

    Periode Data

    Sistem SMART BTQ menggunakan dua jenis periode analisis:

    • Periode Tunggal (1 Bulan)
      Digunakan pada menu berikut:
      • Ringkasan Eksekutif
      • Data Sekolah
      • Analisis Grafik
      • Kesimpulan
      Pada mode ini, data yang ditampilkan berasal dari satu bulan tertentu dan digunakan untuk evaluasi performa pada periode tersebut.
    • Multi Periode (Lebih dari 1 Bulan)
      Digunakan pada menu:
      • Analisis Tren
      Pada mode ini, data dianalisis dalam rentang waktu lebih dari satu bulan untuk melihat tren perkembangan, peningkatan, atau penurunan performa.

    Status Data (Indikator Kualitas)

    Setiap sekolah memiliki status kualitas data sebagai berikut:

    ✔ VALID
    Data Konsisten

    Menunjukkan bahwa data lengkap dan konsisten di dalam database. Semua komponen data (total, lancar, belum, tidak) sesuai dan tidak ada selisih perhitungan.

    ⚠ PARTIAL
    Perlu Perbaikan

    Menunjukkan bahwa data tersedia namun belum memenuhi standar kualitas sepenuhnya.

    Kondisi yang termasuk PARTIAL:
    • Data siswa (total, lancar, belum, tidak) tersedia dan lengkap,
    • namun tingkat kehadiran guru BTQ berada di bawah standar minimum (70%), atau
    • Data guru tidak tersedia.
    Catatan: Data tetap dapat dianalisis, namun memerlukan perhatian untuk peningkatan kualitas.
    ❗ CEK
    Data Tidak Valid

    Menunjukkan bahwa data tidak konsisten atau terdapat kesalahan input.

    Contoh Anomali:
    Total siswa = 20
    Lancar = 10, Belum = 1, Tidak = 1.
    (Jumlah hanya 12, terdapat selisih data yang tidak terisi sehingga dianggap tidak valid).

    Tujuan Sistem

    SMART BTQ dirancang secara spesifik untuk:

    • Monitoring perkembangan kemampuan BTQ siswa secara real-time.
    • Menyediakan analisis berbasis data yang terpusat.
    • Membantu pengambilan keputusan dan kebijakan pembelajaran.
    • Mendeteksi kesalahan dan inkonsistensi data secara otomatis.
